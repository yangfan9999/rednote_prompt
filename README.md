# rednote_prompt
阶段	处理对象	动作	成本控制
第一阶段（爬取）	100 篇 Reddit 帖子	只抓取基础数据（标题、正文、Ups数、评论数、URL）。	极低（纯文本网络请求）
第二阶段（硬过滤）	100 篇 → 20 篇	运行本地代码/轻量脚本：按 Upvote 比例、评论热度、关键词过滤（剔除小红书敏感词、政治、过度本地化的梗）。	零成本
第三阶段（AI 粗筛）	20 篇 → 3 篇	调用大模型（如 GPT-4o-mini 或 Claude Haiku），给一句话 Prompt：“判断以下帖子哪个更符合小红书[你的定位，如：职场/科技/情感]用户的口味，并给出 1-3 篇”。	极低（小模型 Token 极便宜）
第四阶段（精细化生成）	3 篇 → 1 篇	使用主力大模型，进行爆款小红书文风改写（添加 Emoji、提炼金句、设计吸睛标题、规划封面图提示词）。	适中（只对最优质的 1 篇进行重度加工）
